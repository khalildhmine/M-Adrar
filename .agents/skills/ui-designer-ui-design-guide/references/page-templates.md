# Page Templates

> 페이지 유형별 표준 구성 가이드.
> 각 유형의 섹션 순서, 필수/선택 컴포넌트, 레이아웃 스켈레톤, Next.js 파일 구조를 정의한다.

---

## 페이지 유형 빠른 참조

| 유형 | 키워드 | 주요 용도 |
|------|--------|----------|
| Landing | landing, 랜딩, 홈페이지 | 마케팅, 서비스 소개 |
| Dashboard | dashboard, 대시보드 | 데이터 요약, 모니터링 |
| Settings | settings, 설정 | 사용자/시스템 설정 |
| Auth | auth, login, 로그인, 회원가입 | 인증 |
| CRUD List | list, 목록, 게시판 | 데이터 목록 조회 |
| Detail | detail, 상세 | 단일 항목 상세 보기 |
| Pricing | pricing, 요금 | 요금제 비교/선택 |
| Form | form, 폼, 입력 | 데이터 입력/생성 |

## 1. Landing Page

마케팅 중심의 공개 페이지. 사이드바 없이 전체 너비 레이아웃을 사용한다.

### 섹션 순서

| # | 섹션 | 필수/선택 | 설명 |
|---|------|----------|------|
| 1 | Navigation Menu | 필수 | 로고 + 메뉴 링크 + CTA 버튼 |
| 2 | Hero | 필수 | 핵심 가치 제안 + 메인 CTA |
| 3 | Clients / 소셜 프루프 | 선택 | 고객사 로고 롤링 |
| 4 | Features | 필수 | 주요 기능 카드 그리드 |
| 5 | Hooks / 시선 강탈 | 선택 | 강렬한 통계 또는 인용 문구 |
| 6 | Testimonials | 선택 | 실제 사용자 후기 |
| 7 | Comparisons | 선택 | 경쟁사 또는 요금제 간 비교 |
| 8 | Pricing | 선택 | 요금제 카드 |
| 9 | CTA | 필수 | 최종 행동 유도 배너 |
| 10 | Footer | 필수 | 사이트맵 + 법적 고지 |

### 컴포넌트 목록

**필수**
- `NavigationMenu` — 상단 글로벌 내비게이션
- `Hero` — 커스텀 구현 (대형 타이포그래피 + CTA 버튼)
- `Button` — CTA 행동 유도 (size="lg")
- `Footer` — 커스텀 구현 (Separator + 컬럼 그리드)

**선택**
- `Card` — Features 섹션 기능 카드
- `Avatar` + blockquote — Testimonials 후기 카드
- `Badge` — Pricing 추천 플랜 강조
- `Carousel` — Testimonials 슬라이드 방식 사용 시
- `Accordion` — FAQ 섹션 추가 시

### shadcn/ui 매핑

| 목적 | shadcn 컴포넌트 | 비고 |
|------|----------------|------|
| 상단 메뉴 | `NavigationMenu` | 드롭다운 포함 시 |
| CTA 버튼 | `Button` (size="lg") | variant="default" |
| 기능 카드 | `Card`, `CardHeader`, `CardContent` | 아이콘 + 제목 + 설명 |
| 후기 카드 | `Card` + `Avatar` | blockquote 스타일 |
| 요금 배지 | `Badge` | variant="secondary" 추천 플랜 |
| 가격 토글 | `Switch` 또는 `Tabs` | 월간/연간 전환 |
| 로고 롤링 | 커스텀 CSS 마키 | `overflow-hidden` + animation |

### Next.js 파일 구조

```
app/
└── (marketing)/
    ├── layout.tsx          ← 마케팅 전용 레이아웃 (사이드바 없음)
    ├── page.tsx            ← 랜딩 메인 페이지
    └── _components/
        ├── navbar.tsx      ← 상단 내비게이션 바
        ├── hero.tsx        ← 히어로 섹션
        ├── clients.tsx     ← 고객사 로고 롤링
        ├── features.tsx    ← 기능 소개 카드 그리드
        ├── hooks.tsx       ← 시선 강탈 통계 블록
        ├── testimonials.tsx← 사용자 후기
        ├── pricing.tsx     ← 요금제 카드 (월/연 토글 포함)
        ├── cta.tsx         ← 최종 행동 유도 배너
        └── footer.tsx      ← 푸터
```

## 2. Dashboard

데이터 시각화 중심의 내부 관리 페이지. 기존 사이드바 + 헤더 레이아웃을 그대로 사용한다.

### 섹션 순서

| # | 섹션 | 필수/선택 | 설명 |
|---|------|----------|------|
| 1 | Sidebar | 필수 | 기존 앱 레이아웃의 사이드바 |
| 2 | Header | 필수 | 페이지 제목 + 브레드크럼 + 액션 |
| 3 | Stats Cards 행 | 필수 | KPI 요약 카드 (4열 기본) |
| 4 | Charts 그리드 | 선택 | 메인 차트 + 보조 차트/목록 |
| 5 | Data Table | 선택 | 최근 항목 인터랙티브 테이블 |
| 6 | Activity Feed | 선택 | 최근 활동 로그 또는 피드 |

### ASCII 스켈레톤

```
┌──────────┬────────────────────────────────────┐
│          │ Header                             │
│  Side    │ [페이지 제목]      [날짜] [내보내기]│
│  bar     ├────────────────────────────────────┤
│          │                                    │
│  ├─ 홈   │ Stats Cards (4열)                  │
│  ├─ 분석 │ ┌──────┐ ┌──────┐ ┌──────┐ ┌────┐ │
│  ├─ 사용 │ │총매출│ │신규자│ │활성자│ │건수│ │
│  │  자   │ │₩4.5M│ │+2.3K│ │12.2K│ │573 │ │
│  ├─ 설정 │ └──────┘ └──────┘ └──────┘ └────┘ │
│          │                                    │
│          │ Charts (4:3 비율 그리드)            │
│          │ ┌──────────────────┐ ┌──────────┐  │
│          │ │  메인 차트 (4)   │ │보조 (3)  │  │
│          │ │  (라인/바 차트)  │ │(최근 판매)│  │
│          │ └──────────────────┘ └──────────┘  │
│          │                                    │
│          │ Data Table (전체 너비)              │
│          │ ┌──────────────────────────────┐   │
│          │ │ 헤더 | 상태 | 금액 | 날짜    │   │
│          │ │ ─────────────────────────────│   │
│          │ │ 행 1 |  ... |  ... |  ...   │   │
│          │ └──────────────────────────────┘   │
└──────────┴────────────────────────────────────┘
```

### 컴포넌트 목록

**필수**
- `Sidebar` — 기존 앱 사이드바 레이아웃 그대로 사용
- `Card` — KPI 통계 카드 (숫자 + 트렌드 표시)
- `CardHeader`, `CardContent` — 카드 내부 구성

**선택**
- `DataTable` (TanStack Table) — 인터랙티브 데이터 테이블
- `Chart` (recharts/shadcn chart) — 라인, 바, 도넛 차트
- `Badge` — 상태 라벨 (성공/경고/오류)
- `Select` — 날짜 범위 필터 (오늘/이번 주/이번 달)
- `DatePicker` — 커스텀 날짜 범위 선택
- `Skeleton` — 데이터 로딩 중 플레이스홀더

### shadcn/ui 매핑

| 목적 | shadcn 컴포넌트 | 비고 |
|------|----------------|------|
| KPI 카드 | `Card` + `CardHeader` + `CardContent` | 트렌드 Arrow 아이콘 포함 |
| 차트 | `ChartContainer` (shadcn/chart) | recharts 래퍼 |
| 데이터 테이블 | `DataTable` + TanStack Table | 정렬/필터/페이지네이션 |
| 상태 뱃지 | `Badge` | variant로 색상 구분 |
| 날짜 필터 | `Select` 또는 `DatePicker` | Popover + Calendar 조합 |
| 로딩 | `Skeleton` | 카드/테이블 형태 유지 |

### Next.js 파일 구조

```
app/
└── (dashboard)/
    ├── layout.tsx              ← 사이드바 + 헤더 공통 레이아웃
    └── dashboard/
        ├── page.tsx            ← 대시보드 메인 페이지
        └── _components/
            ├── stats-cards.tsx ← KPI 통계 카드 행
            ├── overview-chart.tsx ← 메인 차트
            ├── recent-sales.tsx   ← 보조 패널 (최근 판매 목록)
            └── activity-feed.tsx  ← 최근 활동 피드 (선택)
```

## 3. Settings

카테고리별로 구분된 설정 폼 페이지. Tabs 또는 내부 사이드바로 섹션을 전환한다.

### 섹션 순서

| # | 섹션 | 필수/선택 | 설명 |
|---|------|----------|------|
| 1 | 페이지 헤더 | 필수 | 제목 + 설명 문구 |
| 2 | Tabs 또는 설정 사이드바 | 필수 | 카테고리 내비게이션 |
| 3 | Form Section (일반) | 필수 | 프로필, 계정 등 기본 설정 |
| 4 | Form Section (알림) | 선택 | Switch 토글 목록 |
| 5 | Form Section (보안) | 선택 | 비밀번호 변경, 2FA |
| 6 | Form Section (연동) | 선택 | 소셜/OAuth 연결 상태 |
| 7 | Danger Zone | 선택 | 계정 삭제 등 위험 액션 |
| 8 | Save 버튼 | 필수 | 각 섹션 하단 고정 |

### ASCII 스켈레톤

```
┌──────────┬──────────────────────────────────────┐
│          │ [설정]                               │
│  App     │ 계정 및 환경 설정을 관리합니다.       │
│  Side    ├──────────────────────────────────────┤
│  bar     │                                      │
│          │  ┌──────────────────────────────┐    │
│          │  │ [일반] [알림] [보안] [연동]   │    │  ← Tabs
│          │  └──────────────────────────────┘    │
│          │                                      │
│          │  ┌──────────────────────────────┐    │
│          │  │ 프로필 정보                   │    │
│          │  │ ┌──────────────────────────┐ │    │
│          │  │ │ 이름  [_______________]   │ │    │
│          │  │ │ 이메일[_______________]   │ │    │
│          │  │ │ 소개  [_______________]   │ │    │
│          │  │ └──────────────────────────┘ │    │
│          │  │               [취소] [저장]  │    │
│          │  └──────────────────────────────┘    │

│          │  ┌──────────────────────────────┐    │

│          │  └──────────────────────────────┘    │
└──────────┴──────────────────────────────────────┘
```

### 컴포넌트 목록

**필수**
- `Tabs` — 설정 카테고리 전환 (일반/알림/보안/연동)
- `Form` (react-hook-form + zod) — 각 섹션의 폼
- `Input` — 텍스트 입력 필드
- `Button` — 저장/취소 액션

**선택**
- `Switch` — 알림/기능 On/Off 토글
- `Select` — 언어, 타임존 등 드롭다운 선택
- `Avatar` + `FileUpload` — 프로필 사진 업로드
- `AlertDialog` — 계정 삭제 등 위험 액션 확인 모달
- `Separator` — 섹션 구분선
- `Textarea` — 자기소개 등 장문 입력

### shadcn/ui 매핑

| 목적 | shadcn 컴포넌트 | 비고 |
|------|----------------|------|
| 카테고리 탭 | `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` | |
| 폼 전체 | `Form` | react-hook-form + zod |
| 텍스트 필드 | `FormField` + `Input` | FormLabel, FormMessage 포함 |
| 토글 설정 | `FormField` + `Switch` | |
| 드롭다운 선택 | `FormField` + `Select` | |
| 저장 버튼 | `Button` (type="submit") | 로딩 상태 처리 포함 |
| 위험 구역 | `Button` (variant="destructive") + `AlertDialog` | |
| 섹션 카드 | `Card` + `CardHeader` + `CardContent` + `CardFooter` | |

### Next.js 파일 구조

```
app/
└── (dashboard)/
    └── settings/
        ├── page.tsx              ← 설정 메인 (기본: 일반 탭)
        ├── layout.tsx            ← 설정 전용 레이아웃 (선택)
        └── _components/
            ├── settings-tabs.tsx      ← 탭 내비게이션
            ├── profile-form.tsx       ← 프로필 정보 폼
            ├── notification-form.tsx  ← 알림 설정 토글 목록
            ├── security-form.tsx      ← 비밀번호 변경 폼
            ├── integrations-form.tsx  ← 소셜 연동 상태
            └── danger-zone.tsx        ← 계정 삭제 섹션
```

## 4. Auth (Login / Register)

인증 전용 페이지. 사이드바 없이 센터 정렬 카드 레이아웃을 사용한다.

### 섹션 순서

| # | 섹션 | 필수/선택 | 설명 |
|---|------|----------|------|
| 1 | 로고 | 필수 | 서비스 로고 (카드 상단) |
| 2 | 제목 + 설명 | 필수 | "환영합니다" / "계정 만들기" |
| 3 | 소셜 로그인 버튼 | 선택 | Google, GitHub 등 OAuth |
| 4 | 구분선 (Or) | 선택 | 소셜과 이메일 폼 사이 구분 |
| 5 | 이메일 폼 | 필수 | 이메일 + 비밀번호 필드 |
| 6 | 보조 링크 | 필수 | "비밀번호 찾기" / "회원가입" 링크 |
| 7 | 제출 버튼 | 필수 | "로그인" / "가입하기" |
| 8 | 하단 전환 링크 | 필수 | 로그인↔회원가입 페이지 전환 |

### 컴포넌트 목록

**필수**
- `Form` (react-hook-form + zod) — 전체 폼 상태 및 검증
- `Input` — 텍스트 필드
- `Button` — 취소 + 저장 액션
- `Card` — 폼 섹션 그룹화 (CardHeader + CardContent)

**선택**
- `Select` — 드롭다운 선택 필드
- `Textarea` — 장문 입력 필드
- `Switch` — 토글 설정
- `Checkbox` / `RadioGroup` — 선택형 필드
- `DatePicker` — 날짜 선택 (Popover + Calendar)
- `FileUpload` — 파일/이미지 업로드
- `Stepper` — 멀티스텝 진행 표시 (커스텀 구현)
- `Alert` — 제출 오류 메시지
- `Sonner` (toast) — 저장 완료 알림

## 공통 설계 원칙

### 레이아웃 분리 기준

| 레이아웃 그룹 | 라우트 그룹 | 포함 페이지 |
|-------------|-----------|-----------|
| 사이드바 + 헤더 | `(dashboard)` | Dashboard, Settings, CRUD List, Detail, Form |
| 풀와이드 마케팅 | `(marketing)` | Landing, Pricing |
| 센터 정렬 인증 | `(auth)` | Login, Register, Forgot Password |

### 헤더 액션 버튼 배치 규칙

- 주요 CTA (새로 만들기, 저장)는 항상 **오른쪽**에 배치
- 파괴적 액션 (삭제)은 DropdownMenu 안으로 격리
- 취소 버튼은 저장 버튼 **왼쪽**에 배치 (outline variant)

### 폼 섹션 구성 원칙

- 관련 필드를 `Card`로 묶어 논리적 그룹화
- 필수 필드는 라벨 옆에 `*` 표시 또는 `FormDescription`으로 안내
- 저장 버튼은 폼 하단 고정 또는 sticky 하단 바로 구현
- react-hook-form + zod 조합으로 클라이언트 검증 일관성 유지

### 빈 상태 처리

모든 목록 페이지는 데이터 없음 상태를 반드시 처리한다.

```
빈 상태 구성: [아이콘] + [제목] + [설명] + [CTA 버튼]
예시: [Users 아이콘] + "사용자가 없습니다" + "첫 번째 사용자를 추가해 보세요" + [+ 사용자 추가]
```

### 모바일 반응형 원칙

| 페이지 유형 | 모바일 처리 방식 |
|-----------|----------------|
| Landing | 단일 컬럼 스택, 사이드바 없음 |
| Dashboard | 사이드바 → Sheet 오버레이, 통계 카드 1~2열 |
| Settings | Tabs → Select 드롭다운으로 전환 (선택) |
| Auth | 단일 컬럼, 분할 레이아웃 숨김 |
| CRUD List | 테이블 → 카드 뷰 전환, 가로 스크롤 |
| Detail | 카드 단일 컬럼 스택 |
| Pricing | 카드 단일 컬럼 스택, 추천 플랜 상단 고정 |
| Form | 단일 컬럼, 섹션 카드 스택 |
