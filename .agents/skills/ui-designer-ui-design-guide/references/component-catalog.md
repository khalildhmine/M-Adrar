# UI Component Catalog

> 이 문서는 UI 페이지 설계 시 컴포넌트 선택을 돕는 의사결정 가이드이다.
> 21st.dev 서비스의 컴포넌트 분류를 기반으로 shadcn/ui 매핑을 포함한다.

## 빠른 선택 가이드

| 목적 | 추천 컴포넌트 |
|------|-------------|
| 첫인상/메인 비주얼 | Hero, Background, Navigation Menu |
| 기능 소개 | Features, Images, Comparisons |
| 신뢰 구축 | Testimonials, Clients, Comparisons |
| 행동 유도 | CTA, Pricing, Buttons |
| 데이터 표시 | Tables, Cards, Tabs, Badges |
| 사용자 입력 | Forms, Inputs, Selects, Checkboxes |
| 피드백 | Alerts, Toasts, Notifications, Dialogs |
| 네비게이션 | Navigation Menus, Sidebars, Tabs, Paginations |

---

## Marketing Blocks

### 공지 (Announcements)

- **설명**: 신제품 출시, 이벤트, 점검 등 중요한 소식을 알리는 바 형태의 컴포넌트이다.
- **use-when**:
  - 한정 기간 프로모션이나 할인 쿠폰을 전체 방문자에게 즉시 노출해야 할 때
  - 서비스 점검 예고나 긴급 공지를 가장 눈에 띄는 위치에 배치할 때
  - 신기능 출시를 간결한 한 줄 메시지로 사용자에게 알릴 때
- **avoid-when**:
  - 공지 내용이 길어 한 줄로 요약이 불가능한 경우 (Alert 또는 Dialog 사용)
  - 특정 사용자에게만 보여야 하는 개인화 알림 (Notifications 사용)
- **pairs-with**: Hero, Navigation Menus, CTA
- **shadcn-mapping**: 커스텀 구현 (Alert 컴포넌트 변형 또는 최상단 고정 배너)
- **variants**:
  - 미니멀: 단색 배경에 텍스트+닫기 버튼만 포함한 얇은 바
  - 강조형: 브랜드 컬러 배경에 CTA 링크를 포함한 두꺼운 바
  - 다크: 검정 배경에 흰 텍스트로 고급스러운 공지 표시

---

### 배경 (Backgrounds)

- **설명**: 섹션의 뒤쪽에 위치하여 분위기를 결정하는 요소이다. 단색, 그라데이션, 패턴 등이 포함된다.
- **use-when**:
  - Hero나 CTA 섹션에 시각적 깊이감을 더해 사용자 시선을 집중시킬 때
  - 콘텐츠 블록 간 영역 구분을 색상이나 텍스처로 명확히 해야 할 때
  - 브랜드 아이덴티티를 강화하기 위해 독특한 시각적 분위기를 연출할 때
- **avoid-when**:
  - 배경이 본문 텍스트의 가독성을 해치는 경우 (충분한 대비율 확보 필수)
  - 성능에 민감한 페이지에서 고해상도 배경 이미지를 과도하게 사용하는 경우
- **pairs-with**: Hero, Shaders, Features, CTA
- **shadcn-mapping**: 커스텀 구현 (CSS 또는 Tailwind 유틸리티 클래스 활용)
- **variants**:
  - 미니멀: 단색 또는 미세한 노이즈 텍스처
  - 그라데이션: 두 가지 브랜드 컬러 간 부드러운 그라데이션
  - 인터랙티브: 마우스 이동에 반응하는 파티클 또는 mesh gradient

---

### 테두리/구분선 (Borders)

- **설명**: 콘텐츠 블록 간의 경계를 나누거나 강조할 때 사용하는 디자인 요소이다.
- **use-when**:
  - 섹션과 섹션 사이를 명확히 구분하여 스크롤 흐름을 정리할 때
  - 카드나 컨테이너의 외곽선으로 요소의 경계를 정의할 때
  - 장식적 구분선(divider)으로 콘텐츠 흐름에 리듬감을 줄 때
- **avoid-when**:
  - 이미 여백(padding/margin)으로 충분히 구분되는 곳에 중복 사용하는 경우
  - 과도한 테두리 중첩으로 인해 UI가 산만해지는 경우
- **pairs-with**: Cards, Tables, Features, Texts
- **shadcn-mapping**: Separator
- **variants**:
  - 미니멀: 1px 실선, 저채도 컬러
  - 장식형: 그라데이션 또는 점선으로 섹션 구분 강조
  - 다크: 배경과 유사한 어두운 계열의 미묘한 구분선

---

### CTA/구매 유도 (Calls to Action)

- **설명**: 사용자의 클릭이나 특정 행동(가입, 구매)을 유도하는 버튼과 문구 세트이다.
- **use-when**:
  - 랜딩 페이지 핵심 전환 지점에서 가입·구매를 직접 유도할 때
  - Features나 Pricing 섹션 하단에서 행동 전환을 마무리할 때
  - 스크롤 중간에 사용자의 관심이 최고조에 달하는 시점에 배치할 때
- **avoid-when**:
  - 페이지 내 CTA가 3개 이상 반복되어 사용자가 혼란을 겪는 경우
  - 아직 가치를 충분히 전달하지 않은 상태에서 너무 이른 시점에 배치하는 경우
- **pairs-with**: Hero, Pricing, Features, Hooks
- **shadcn-mapping**: Button (variant: default, size: lg)
- **variants**:
  - 단순형: 큰 버튼 하나 + 보조 텍스트 링크
  - 이중형: 주 CTA 버튼 + 보조 CTA 버튼 나란히 배치
  - 배너형: 전체 너비 배경색 섹션에 문구와 버튼을 중앙 정렬

---

### 고객사 (Clients)

- **설명**: 해당 서비스를 사용하는 유명 기업들의 로고를 모아 보여주는 섹션이다.
- **use-when**:
  - B2B 서비스에서 잠재 고객의 신뢰도를 높이기 위해 레퍼런스를 공개할 때
  - Hero 섹션 바로 아래에서 첫인상 직후 신뢰감을 즉시 보강할 때
  - 브랜드 인지도가 높은 파트너사가 3개 이상 확보된 경우
- **avoid-when**:
  - 초기 스타트업으로 유명 고객사 로고가 없는 경우 (Testimonials로 대체)
  - B2C 소비자 대상 서비스에서 기업 로고 나열이 어색한 경우
- **pairs-with**: Hero, Testimonials, Features
- **shadcn-mapping**: 커스텀 구현 (로고 그리드 또는 무한 스크롤 마키 레이아웃)
- **variants**:
  - 정적 그리드: 로고를 격자 형태로 배치
  - 마키 스크롤: 로고가 좌우로 자동 흐르는 애니메이션 띠
  - 회색조: 컬러 로고를 grayscale 처리하여 일관된 톤 유지

---

### 비교 (Comparisons)

- **설명**: 자사 서비스의 요금제 차이나 경쟁사와의 기능을 한눈에 비교하는 표이다.
- **use-when**:
  - Pricing 페이지에서 요금제별 포함 기능 차이를 명확히 보여줄 때
  - 경쟁사 대비 자사 서비스의 우위를 객관적 항목으로 제시할 때
  - Before/After 또는 구버전/신버전 차이를 시각적으로 비교할 때
- **avoid-when**:
  - 비교 항목이 3개 미만으로 표가 너무 단순한 경우 (Badges나 Features로 대체)
  - 경쟁사 정보가 부정확하거나 업데이트가 어려운 경우
- **pairs-with**: Pricing, Features, Tables, Badges
- **shadcn-mapping**: Table + Badge 조합, 또는 커스텀 비교 카드
- **variants**:
  - 체크리스트형: 행마다 체크/엑스 아이콘으로 포함 여부 표시
  - 슬라이더형: 두 이미지를 겹쳐 드래그로 비교하는 인터랙티브 방식
  - 하이라이트형: 추천 요금제 컬럼을 색상으로 강조한 3단 비교표

---

### 독/고정 바 (Docks)

- **설명**: 화면 하단 등에 고정되어 주요 메뉴로 바로 이동하게 돕는 플로팅 바이다.
- **use-when**:
  - 모바일 웹에서 하단 네비게이션을 항상 접근 가능하게 유지할 때
  - 긴 페이지에서 주요 액션(홈, 검색, 마이페이지)을 빠르게 실행할 때
  - 앱 스타일 UX를 웹에서 구현할 때
- **avoid-when**:
  - 데스크탑 전용 관리자 대시보드 (Sidebar로 대체)
  - 콘텐츠가 하단에 위치하여 독이 가리는 레이아웃
- **pairs-with**: Navigation Menus, Sidebars, Scroll Areas
- **shadcn-mapping**: 커스텀 구현 (fixed position + Tooltip 조합)
- **variants**:
  - 미니멀: 아이콘만 나열, 라벨 없음
  - 라벨형: 아이콘 아래 텍스트 라벨 포함
  - 플로팅: 배경에서 살짝 떠 있는 둥근 모서리 바

---

### 기능 소개 (Features)

- **설명**: 제품이 가진 주요 장점이나 기능을 아이콘과 함께 설명하는 블록이다.
- **use-when**:
  - 랜딩 페이지에서 제품의 핵심 가치를 3~6가지로 정리하여 보여줄 때
  - Hero 섹션 이후 구체적인 기능 설명으로 자연스럽게 연결할 때
  - 복잡한 제품을 단순화된 카드형 설명으로 이해시킬 때
- **avoid-when**:
  - 기능이 1~2개뿐인 단순한 서비스 (텍스트 블록으로 충분)
  - 기능 설명 없이 아이콘만 나열하는 의미 없는 장식용 배치
- **pairs-with**: Hero, CTA, Images, Comparisons
- **shadcn-mapping**: Card + Badge 조합
- **variants**:
  - 그리드형: 3열 또는 4열 아이콘+제목+설명 카드 배열
  - 교대형: 이미지와 텍스트가 좌우 교대로 배치되는 레이아웃
  - 타임라인형: 단계별 기능을 세로 흐름으로 설명

---

### 푸터 (Footers)

- **설명**: 웹사이트의 최하단 영역으로 기업 정보, 약관, SNS 링크 등이 담긴다.
- **use-when**:
  - 모든 페이지의 공통 하단 영역으로 법적 고지, 사이트맵, 연락처를 제공할 때
  - 브랜드 로고와 간단한 소개를 하단에서 마지막으로 각인시킬 때
  - SNS, 뉴스레터 구독 링크 등 부가적인 연결 경로를 제공할 때
- **avoid-when**:
  - 랜딩 페이지 원페이지 구성에서 지나치게 상세한 푸터가 이탈을 유발하는 경우
  - 모바일에서 링크 수가 너무 많아 스크롤이 과도하게 길어지는 경우
- **pairs-with**: Navigation Menus, Links, Scroll Areas
- **shadcn-mapping**: 커스텀 구현 (Separator + 컬럼 레이아웃)
- **variants**:
  - 미니멀: 저작권 문구 한 줄 + 링크 몇 개
  - 풍성형: 4~5컬럼 사이트맵 + 뉴스레터 구독 폼
  - 다크: 검정 배경에 흰 텍스트로 페이지 전체와 대비

---

### 히어로 섹션 (Heroes)

- **설명**: 웹사이트 접속 시 가장 먼저 보이는 거대한 상단 영역이다.
- **use-when**:
  - 서비스의 핵심 가치 제안을 단 한 문장으로 전달하는 첫인상을 만들 때
  - 강렬한 비주얼과 함께 즉각적인 CTA로 전환을 유도할 때
  - 브랜드 스토리텔링의 시작점으로 감정적 연결을 형성할 때
- **avoid-when**:
  - 로그인 후 대시보드처럼 첫인상보다 기능이 중요한 내부 페이지
  - 정보 밀도가 높아야 하는 검색 결과 페이지나 목록 페이지
- **pairs-with**: Announcements, Navigation Menus, CTA, Backgrounds
- **shadcn-mapping**: 커스텀 구현 (Button + 타이포그래피 구성)
- **variants**:
  - 텍스트 중심: 큰 헤드라인 + 부제 + CTA 버튼 (배경 단색)
  - 이미지/영상형: 전체 화면 배경 미디어 + 오버레이 텍스트
  - 분할형: 좌측 텍스트 + 우측 제품 목업 이미지 나란히 배치

---

### 훅/시선 강탈 (Hooks)

- **설명**: 사용자의 호기심을 자극하거나 강력한 문구로 이목을 끄는 짧은 문구 블록이다.
- **use-when**:
  - 섹션과 섹션 사이 전환 구간에서 스크롤을 멈추게 하는 강렬한 한 문장이 필요할 때
  - 기능 나열 후 감정적 공감을 이끌어내는 브리지 역할이 필요할 때
  - 숫자나 통계를 활용한 임팩트 있는 수치 강조가 필요할 때
- **avoid-when**:
  - 이미 충분히 강조된 Hero 바로 다음에 연달아 배치하는 경우 (강조 피로도 유발)
  - 근거 없는 과장 문구로 신뢰를 잃을 수 있는 경우
- **pairs-with**: Features, CTA, Testimonials, Texts
- **shadcn-mapping**: 커스텀 구현 (대형 타이포그래피 블록)
- **variants**:
  - 수치형: "3초 만에", "98% 만족도" 등 숫자를 크게 강조
  - 질문형: 사용자에게 직접 질문을 던지는 인터랙티브 문구
  - 인용형: 업계 유명인의 말을 인용하여 권위를 부여

---

### 이미지 (Images)

- **설명**: 제품 사진, 그래픽 디자인 등 시각적 자료를 배치하는 레이아웃이다.
- **use-when**:
  - 텍스트만으로 설명이 부족한 제품 UI나 실제 사용 장면을 보여줄 때
  - Features 섹션에서 각 기능의 시각적 증거로 스크린샷을 제시할 때
  - 감성적 브랜딩을 위해 고화질 라이프스타일 사진을 배치할 때
- **avoid-when**:
  - 최적화되지 않은 대용량 이미지를 페이지 로딩 성능을 희생하며 삽입하는 경우
  - 이미지가 콘텐츠의 맥락과 무관한 스톡 사진 남용
- **pairs-with**: Features, Carousels, Heroes, Backgrounds
- **shadcn-mapping**: Carousel (슬라이드 갤러리), 또는 커스텀 이미지 그리드
- **variants**:
  - 단독형: 전체 너비 또는 큰 단일 이미지
  - 갤러리형: 다수의 이미지를 격자 또는 마소닉 레이아웃으로 배치
  - 목업형: 디바이스 프레임 안에 제품 스크린샷을 삽입한 형태

---

### 지도 (Maps)

- **설명**: 오프라인 매장이나 사무실의 위치를 보여주는 대화형 지도 블록이다.
- **use-when**:
  - 오프라인 매장, 사무실, 행사 장소의 위치를 방문자에게 안내할 때
  - '오시는 길' 페이지에서 대중교통 접근 정보와 함께 제공할 때
  - 다수의 지점을 핀으로 표시하는 멀티 로케이션 서비스에서
- **avoid-when**:
  - 순수 디지털 서비스로 물리적 위치 정보가 불필요한 경우
  - 지도 API 비용이나 개인정보 정책 이슈가 있는 경우
- **pairs-with**: Footers, Texts, Forms
- **shadcn-mapping**: 커스텀 구현 (Google Maps, Naver Maps, Kakao Maps iframe 또는 SDK 삽입)
- **variants**:
  - 임베드형: iframe으로 지도 전체 삽입
  - 썸네일형: 정적 지도 이미지 + 외부 지도 링크 버튼
  - 인터랙티브형: 핀, 클러스터링 등 커스텀 마커가 있는 전용 지도

---

### 네비게이션 메뉴 (Navigation Menus)

- **설명**: 사용자가 원하는 페이지로 이동할 수 있게 돕는 상단 메뉴 바이다.
- **use-when**:
  - 웹사이트 최상단(GNB)에서 주요 페이지 간 이동 경로를 제공할 때
  - 드롭다운 메가메뉴로 복잡한 사이트 구조를 계층적으로 노출할 때
  - 로고, 메뉴, CTA 버튼을 하나의 일관된 헤더로 구성할 때
- **avoid-when**:
  - 메뉴 항목이 8개 이상으로 과도하게 많아 인지 부하를 주는 경우
  - 원페이지 랜딩에서 앵커 링크만 필요한 경우 (단순 앵커 메뉴 사용)
- **pairs-with**: Announcements, Hero, Sidebars, Footers
- **shadcn-mapping**: NavigationMenu
- **variants**:
  - 투명형: 히어로 위에 겹쳐 배경이 투명한 상태, 스크롤 시 불투명 전환
  - 고정형: 스크롤해도 항상 상단에 고정되는 sticky 헤더
  - 메가메뉴형: 호버 시 넓은 드롭다운 패널이 펼쳐지는 다단 메뉴

---

### 가격 섹션 (Pricing Sections)

- **설명**: 유료 서비스의 구독료나 구매 가격을 카드 형태로 보여주는 블록이다.
- **use-when**:
  - SaaS 서비스의 요금제를 월간/연간 단위로 비교하여 선택을 유도할 때
  - 무료 플랜과 유료 플랜의 기능 차이를 명확히 제시할 때
  - 연간 결제 할인 혜택을 강조하여 장기 구독을 유도할 때
- **avoid-when**:
  - 가격이 견적 문의 방식으로 공개가 불가한 엔터프라이즈 전용 서비스
  - 단일 상품으로 가격 비교가 무의미한 경우
- **pairs-with**: Comparisons, CTA, Testimonials, Badges
- **shadcn-mapping**: Card + Badge + Button + Toggle (월간/연간 전환)
- **variants**:
  - 3단 카드형: 베이직/스탠다드/프로 3개 플랜을 나란히 배치
  - 하이라이트형: 추천 플랜을 더 크거나 다른 색으로 강조
  - 비교 테이블 연계형: 가격 카드 하단에 상세 기능 비교표 연결

---

### 스크롤 영역 (Scroll Areas)

- **설명**: 가로나 세로로 스크롤하여 더 많은 정보를 볼 수 있게 만든 영역이다.
- **use-when**:
  - 많은 수의 카드나 이미지를 좁은 공간에서 가로 스크롤로 탐색할 때
  - 모바일에서 테이블처럼 넓은 콘텐츠를 가로 스크롤로 처리할 때
  - 뉴스레터, 상품 목록 등을 무한 스크롤 방식으로 제공할 때
- **avoid-when**:
  - 가로 스크롤이 사용자에게 예측 불가능한 경험을 줄 수 있는 메인 레이아웃
  - 전체 페이지 스크롤과 중첩되어 혼란을 주는 경우
- **pairs-with**: Cards, Carousels, Tables, Images
- **shadcn-mapping**: ScrollArea
- **variants**:
  - 수평 스크롤: 카드 목록을 좌우로 드래그하여 탐색
  - 수직 스크롤: 패널 내부에서만 스크롤되는 고정 높이 영역
  - 스냅 스크롤: 항목별로 딱 맞게 정지하는 snap scroll 방식

---

### 쉐이더/효과 (Shaders)

- **설명**: 배경이나 이미지 위에 덧씌우는 그림자, 빛, 질감 등의 특수 효과이다.
- **use-when**:
  - 이미지 하단을 어둡게 처리하여 텍스트 가독성을 확보해야 할 때
  - 섹션 경계에 부드러운 페이드 효과로 자연스러운 전환을 만들 때
  - 3D 질감이나 빛 산란 효과로 고급스러운 비주얼을 연출할 때
- **avoid-when**:
  - 단순한 콘텐츠 페이지에서 불필요한 시각적 복잡성을 더하는 경우
  - 성능에 민감한 모바일 환경에서 GPU 집약적 효과를 남용하는 경우
- **pairs-with**: Backgrounds, Heroes, Images, Videos
- **shadcn-mapping**: 커스텀 구현 (CSS backdrop-filter, WebGL, Three.js 등)
- **variants**:
  - 그라데이션 오버레이: 이미지 위에 반투명 색상 레이어
  - 노이즈 텍스처: 미세한 필름 그레인 효과로 깊이감 추가
  - 글로우/블러: 특정 요소 주변을 빛나게 하거나 흐리게 처리

---

### 후기 (Testimonials)

- **설명**: 실제 사용자들의 추천사나 별점 리뷰를 모아 놓은 섹션이다.
- **use-when**:
  - Pricing 섹션 직전에서 구매 결정을 앞둔 사용자의 불안을 해소할 때
  - 실제 고객의 목소리로 마케팅 문구보다 신뢰도 높은 증거를 제시할 때
  - 다양한 사용자 유형의 후기로 여러 타겟 고객에게 공감을 이끌 때
- **avoid-when**:
  - 가짜 또는 확인되지 않은 후기를 신뢰성 없이 나열하는 경우
  - 후기 내용이 지나치게 광고성이어서 오히려 불신을 유발하는 경우
- **pairs-with**: Clients, Pricing, CTA, Avatars
- **shadcn-mapping**: Card + Avatar + Badge (별점) 조합
- **variants**:
  - 그리드형: 여러 후기 카드를 격자로 배치
  - 캐러셀형: 하나씩 슬라이드로 넘겨보는 방식
  - 마소닉형: 다양한 길이의 후기를 핀터레스트 스타일로 배치

---

### 텍스트 블록 (Texts)

- **설명**: 긴 본문, 제목, 설명 등 글자로만 이루어진 다양한 서식의 블록이다.
- **use-when**:
  - 블로그 포스트, 이용 약관, 가이드 문서 등 장문의 글이 필요할 때
  - Features 섹션의 각 항목에 상세 설명을 추가할 때
  - 브랜드 스토리나 회사 소개를 서사적으로 전달할 때
- **avoid-when**:
  - 정보를 시각화할 수 있는데 텍스트만 나열하여 이해를 어렵게 하는 경우
  - 모바일에서 읽기 불편한 과도하게 긴 단락
- **pairs-with**: Images, Hooks, Borders, Accordions
- **shadcn-mapping**: 커스텀 구현 (타이포그래피 유틸리티 클래스 활용)
- **variants**:
  - 본문형: 표준 읽기 너비(65~75자)를 유지하는 아티클 레이아웃
  - 강조형: 대형 제목과 소형 본문의 극적인 크기 대비
  - 인용형: blockquote 스타일로 핵심 문장을 시각적으로 분리

---

### 비디오 (Videos)

- **설명**: 홍보 영상이나 튜토리얼 영상을 재생할 수 있는 블록이다.
- **use-when**:
  - 텍스트나 이미지로 설명하기 어려운 제품 사용 방법을 시연할 때
  - 브랜드 필름으로 감성적 연결을 강화할 때
  - 사용자 인터뷰 영상으로 Testimonials를 강화할 때
- **avoid-when**:
  - 자동 재생 + 소리로 사용자 경험을 방해하는 경우 (반드시 음소거 기본값)
  - 인터넷 속도가 느린 환경에서 고용량 영상을 최적화 없이 직접 삽입하는 경우
- **pairs-with**: Heroes, Features, Images, CTA
- **shadcn-mapping**: 커스텀 구현 (HTML5 video 또는 YouTube/Vimeo 임베드)
- **variants**:
  - 임베드형: YouTube/Vimeo iframe을 썸네일+재생 버튼으로 래핑
  - 배경형: Hero 섹션 배경으로 자동 재생되는 음소거 루프 영상
  - 라이트박스형: 썸네일 클릭 시 모달에서 전체 화면으로 재생

## UI Components

### 아코디언 (Accordions)

- **설명**: 클릭하면 내용이 아래로 펼쳐지고 다시 클릭하면 접히는 컴포넌트이다.
- **use-when**:
  - FAQ 페이지에서 많은 질문을 한 화면에 압축하여 보여줄 때
  - 설정 페이지의 카테고리별 옵션을 접어서 정리할 때
  - 모바일에서 긴 콘텐츠를 공간 효율적으로 표시할 때
- **avoid-when**:
  - 항목이 3개 미만으로 펼침 동작이 불필요하게 느껴지는 경우
  - 사용자가 항목을 비교하며 봐야 하는 경우 (펼쳐도 한 번에 하나만 보임)
- **pairs-with**: Forms, Sidebars, Tabs, Cards
- **shadcn-mapping**: Accordion
- **variants**:
  - 단일 열림: 한 번에 하나의 항목만 열리는 방식
  - 다중 열림: 여러 항목을 동시에 열 수 있는 방식
  - 테두리형: 각 항목에 구분선 또는 테두리가 있는 구조화된 형태

---

### AI 채팅 (AI Chats)

- **설명**: 사용자와 인공지능이 대화를 주고받을 수 있는 채팅창 인터페이스이다.
- **use-when**:
  - 고객 지원 챗봇으로 FAQ 자동 응답이나 상담 접수를 처리할 때
  - AI 비서 서비스에서 자연어로 명령을 입력받을 때
  - 제품 내 AI 기능의 메인 인터페이스로 활용할 때
- **avoid-when**:
  - AI 백엔드 없이 단순 규칙 기반 응답만 하는 경우 (일반 채팅 UI로 대체)
  - 복잡한 폼 입력이 더 효율적인 경우에 대화형 UI를 강요하는 경우
- **pairs-with**: Inputs, Buttons, Toasts, Notifications
- **shadcn-mapping**: 커스텀 구현 (ScrollArea + Input + Button 조합)
- **variants**:
  - 사이드패널형: 페이지 우측에 서랍처럼 열리는 채팅창
  - 전체화면형: 채팅이 메인 콘텐츠인 전용 페이지 레이아웃
  - 플로팅형: 화면 우하단에 떠 있는 최소화/최대화 가능한 채팅 버블

---

### 알림 창 (Alerts)

- **설명**: 사용자에게 중요한 정보나 경고, 성공 메시지를 전달하는 피드백 박스이다.
- **use-when**:
  - 폼 제출 결과(성공/실패)를 인라인으로 즉시 알릴 때
  - 시스템 오류, 네트워크 단절 등 중요한 상태 변화를 페이지 내에서 안내할 때
  - 중요한 경고(삭제 불가 복구 없음 등)를 행동 전에 미리 고지할 때
- **avoid-when**:
  - 일시적인 성공 알림 (자동으로 사라지는 Toast 사용)
  - 사용자 확인이 필요한 중요 결정 (AlertDialog 또는 Dialog 사용)
- **pairs-with**: Forms, Buttons, Dialogs, Toasts
- **shadcn-mapping**: Alert (variant: default, destructive)
- **variants**:
  - 정보형: 파란색, 안내 아이콘 포함
  - 경고형: 노란색, 주의 아이콘 포함
  - 에러형: 빨간색, 오류 아이콘 포함
  - 성공형: 초록색, 체크 아이콘 포함

---

### 아바타 (Avatars)

- **설명**: 사용자의 프로필 사진이나 이니셜을 원형 또는 사각형으로 보여주는 이미지 요소이다.
- **use-when**:
  - 댓글, 채팅, 피드에서 작성자를 시각적으로 구분할 때
  - 팀원 목록이나 멤버 목록에서 사람을 나타낼 때
  - 마이페이지 헤더에서 현재 로그인 사용자를 표시할 때
- **avoid-when**:
  - 프로필 이미지가 없을 때 빈 원을 그대로 노출하는 경우 (이니셜 또는 기본 아이콘으로 대체)
  - 아바타 크기가 너무 작아 인식이 불가능한 경우
- **pairs-with**: Testimonials, Notifications, Menus, Cards
- **shadcn-mapping**: Avatar
- **variants**:
  - 원형: 표준 원형 아바타
  - 스택형: 여러 아바타가 겹쳐 쌓인 그룹 표시 (예: +3명)
  - 상태 뱃지형: 온라인/오프라인 상태 점이 아바타 우하단에 표시

---

### 배지 (Badges)

- **설명**: 특정 상태나 숫자를 나타내기 위해 요소 옆에 붙는 작은 태그이다.
- **use-when**:
  - 읽지 않은 알림 개수나 장바구니 아이템 수를 아이콘 위에 표시할 때
  - 제품의 상태(신규, 품절, 할인)를 카드에 표시할 때
  - 사용자 역할(관리자, 프리미엄 회원)을 라벨로 구분할 때
- **avoid-when**:
  - 중요한 정보를 배지 하나로만 전달하여 놓치기 쉬운 경우
  - 배지가 너무 많아 UI가 산만해지는 경우
- **pairs-with**: Notifications, Cards, Avatars, Tables
- **shadcn-mapping**: Badge
- **variants**:
  - 카운터형: 숫자를 표시하는 알림 뱃지
  - 상태형: 색상으로 상태를 구분 (초록/노랑/빨강)
  - 라벨형: 텍스트 키워드를 캡슐 형태로 표시

---

### 버튼 (Buttons)

- **설명**: 사용자가 클릭하여 특정 동작을 실행하게 하는 가장 기본적인 요소이다.
- **use-when**:
  - 폼 제출, 페이지 이동, 모달 열기 등 모든 사용자 액션의 트리거
  - CTA 섹션에서 주요 전환 행동을 명확히 유도할 때
  - 위험한 작업(삭제, 취소) 전 명시적인 의도 확인이 필요할 때
- **avoid-when**:
  - 단순 페이지 이동에 버튼을 사용하는 경우 (Links 사용이 시맨틱에 적합)
  - 비활성 상태의 버튼을 이유 없이 숨기는 경우 (disabled 상태로 이유 표시)
- **pairs-with**: Forms, Dialogs, CTA, Inputs
- **shadcn-mapping**: Button (variant: default, outline, ghost, destructive, link)
- **variants**:
  - 주요(Primary): 가장 강조된 색상, 페이지당 1개 원칙
  - 보조(Secondary/Outline): 주요 버튼 옆 대안 행동
  - 유령(Ghost): 배경 없는 텍스트형, 덜 강조된 액션

---

### 캘린더 (Calendars)

- **설명**: 날짜를 확인하거나 특정 날짜의 일정을 관리할 수 있는 달력 형태의 컴포넌트이다.
- **use-when**:
  - 예약 시스템에서 날짜 선택 UI가 직관적으로 필요할 때
  - 일정 관리 대시보드에서 이벤트를 달력 위에 시각화할 때
  - 특정 기간을 선택하는 날짜 범위 선택기로 활용할 때
- **avoid-when**:
  - 단순히 날짜 하나를 입력받을 때 (Date Picker로 충분)
  - 모바일에서 달력 전체를 표시하면 너무 작아 사용이 어려운 경우
- **pairs-with**: Date Pickers, Forms, Cards, Tables
- **shadcn-mapping**: Calendar
- **variants**:
  - 월간 뷰: 한 달 전체를 격자로 표시
  - 범위 선택: 시작일~종료일을 한 번에 선택 가능한 듀얼 캘린더
  - 이벤트형: 날짜에 이벤트 닷(dot)이나 배지를 표시

---

### 카드 (Cards)

- **설명**: 이미지, 제목, 본문을 하나의 박스 안에 담아 정보를 그룹화한 레이아웃이다.
- **use-when**:
  - 대시보드에서 KPI 수치나 요약 정보를 각각의 블록으로 표현할 때
  - 콘텐츠 목록(블로그, 상품, 팀원)을 균일한 형태로 나열할 때
  - 기능 소개 섹션에서 각 기능을 독립적인 카드로 구성할 때
- **avoid-when**:
  - 단순 텍스트 나열에 불필요한 카드 박스를 남발하는 경우
  - 카드 내부에 너무 많은 정보를 욱여넣어 가독성을 해치는 경우
- **pairs-with**: Badges, Buttons, Avatars, Carousels, Tables
- **shadcn-mapping**: Card (CardHeader, CardContent, CardFooter)
- **variants**:
  - 기본형: 흰 배경 + 그림자 + 테두리
  - 수평형: 이미지 좌측 + 콘텐츠 우측 배치
  - 글래스모피즘: 반투명 배경 + blur 효과

---

### 캐러셀 (Carousels)

- **설명**: 여러 개의 배너나 이미지를 좌우 슬라이드로 넘겨볼 수 있는 컴포넌트이다.
- **use-when**:
  - 홈페이지 상단 메인 배너를 자동 또는 수동으로 전환할 때
  - 제품 상세 페이지에서 여러 각도의 제품 이미지를 슬라이드로 볼 때
  - 추천 콘텐츠 목록을 좁은 영역에서 넘겨볼 수 있게 제공할 때
- **avoid-when**:
  - 자동 전환 캐러셀에서 중요한 CTA를 배치하는 경우 (사용자가 놓칠 수 있음)
  - 모바일에서 스와이프 제스처가 스크롤과 충돌하는 경우
- **pairs-with**: Images, Cards, Heroes, Scroll Areas
- **shadcn-mapping**: Carousel
- **variants**:
  - 전체 너비형: 화면 전체를 덮는 메인 배너 슬라이더
  - 카드 캐러셀: 카드 여러 개가 옆으로 보이는 부분 노출형
  - 자동 재생형: 일정 시간마다 자동으로 넘어가는 방식

---

### 체크박스 (Checkboxes)

- **설명**: 여러 항목 중 하나 이상을 중복 선택할 때 사용하는 선택 상자이다.
- **use-when**:
  - 약관 동의처럼 사용자가 명시적으로 확인해야 하는 항목에 사용
  - 필터 설정에서 여러 카테고리를 동시에 선택할 때
  - 관리자 페이지에서 여러 항목을 한 번에 선택하여 일괄 처리할 때
- **avoid-when**:
  - 단 하나만 선택 가능한 경우 (Radio Group 사용)
  - 켜기/끄기 상태 전환의 경우 (Toggle/Switch 사용)
- **pairs-with**: Forms, Tables, Radio Groups, Buttons
- **shadcn-mapping**: Checkbox
- **variants**:
  - 단독형: 약관 동의처럼 라벨과 함께 단독 사용
  - 목록형: 세로로 나열된 다중 선택 필터 목록
  - 전체 선택형: 테이블 헤더의 전체 선택/해제 체크박스

---

### 날짜 선택기 (Date Pickers)

- **설명**: 날짜를 입력하거나 달력 팝업에서 날짜를 선택하게 돕는 입력창이다.
- **use-when**:
  - 생년월일, 예약 날짜 등 정확한 날짜 입력이 필요한 폼 필드에 사용
  - 날짜 형식 오류를 방지하기 위해 직접 타이핑 대신 선택을 유도할 때
  - 여행 기간처럼 시작일과 종료일을 함께 선택할 때
- **avoid-when**:
  - 연도만 선택하면 되는 경우 (Select로 단순화)
  - 날짜 범위가 매우 넓거나 특정 날짜를 빠르게 탐색해야 할 때 (Calendar 사용)
- **pairs-with**: Forms, Calendars, Inputs, Buttons
- **shadcn-mapping**: DatePicker (Popover + Calendar 조합)
- **variants**:
  - 단일 날짜: 하나의 날짜만 선택
  - 날짜 범위: 시작일과 종료일을 함께 선택
  - 날짜+시간: 날짜와 시간을 함께 선택하는 DateTimePicker

---

### 모달/팝업 (Dialogs / Modals)

- **설명**: 현재 화면 위에 겹쳐서 나타나며, 사용자의 확인이나 입력을 요구하는 창이다.
- **use-when**:
  - 삭제, 로그아웃 등 되돌릴 수 없는 작업 전 사용자 확인을 받을 때
  - 회원가입, 로그인 등 현재 컨텍스트를 벗어나지 않고 폼을 표시할 때
  - 이미지 확대, 상세 정보 등 부가 콘텐츠를 오버레이로 보여줄 때
- **avoid-when**:
  - 사용자가 완료하지 않고 닫기 어려운 복잡한 멀티스텝 프로세스 (전용 페이지 사용)
  - 모바일에서 화면을 거의 다 덮어 불편한 경우 (Drawer/Sheet로 대체)
- **pairs-with**: Buttons, Forms, Alerts, Drawers
- **shadcn-mapping**: Dialog, AlertDialog
- **variants**:
  - 확인형: 제목 + 설명 + 확인/취소 버튼만 있는 최소형
  - 폼형: 내부에 입력 폼이 포함된 데이터 입력 모달
  - 미디어형: 이미지나 영상을 전체 화면에 가깝게 보여주는 라이트박스

---

### 드롭다운 (Dropdowns)

- **설명**: 클릭하면 선택 가능한 목록이 아래로 펼쳐지는 컴포넌트이다.
- **use-when**:
  - 네비게이션 메뉴에서 하위 항목을 펼쳐 보여줄 때
  - 테이블 행의 액션 버튼(편집, 삭제, 복사)을 메뉴로 묶을 때
  - 사용자 프로필 아이콘 클릭 시 계정 관련 옵션을 제공할 때
- **avoid-when**:
  - 선택지가 5개 미만으로 Radio Group이나 Tab이 더 명확한 경우
  - 긴 목록에서 검색 기능이 필요한 경우 (Command/Combobox 사용)
- **pairs-with**: Navigation Menus, Tables, Buttons, Menus
- **shadcn-mapping**: DropdownMenu
- **variants**:
  - 액션 메뉴형: 아이콘 버튼 클릭 시 편집/삭제 등 액션 목록
  - 네비게이션형: 메뉴 항목 호버 시 하위 메뉴 펼침
  - 체크박스형: 드롭다운 내 항목에 체크 표시로 현재 상태 표시

---

### 빈 화면 (Empty States)

- **설명**: 데이터나 검색 결과가 없을 때 보여주는 안내 화면이다.
- **use-when**:
  - 장바구니, 알림 목록 등이 비어 있을 때 안내와 함께 행동을 유도할 때
  - 검색 결과가 없을 때 검색어 수정이나 다른 행동을 제안할 때
  - 신규 사용자가 처음 접했을 때 온보딩을 자연스럽게 유도할 때
- **avoid-when**:
  - 로딩 중인 상태를 빈 화면으로 잘못 표시하는 경우 (Skeleton/Spinner 사용)
  - 에러 상태를 빈 화면으로 숨기는 경우 (Alert로 명확히 안내)
- **pairs-with**: Tables, Cards, Alerts, Buttons
- **shadcn-mapping**: 커스텀 구현 (SVG 일러스트 + 타이포그래피 + Button 조합)
- **variants**:
  - 일러스트형: 귀여운 또는 브랜드 일러스트 + 안내 문구
  - 아이콘형: 단순 아이콘 + 짧은 설명 + CTA 버튼
  - 검색 결과 없음형: 검색어를 강조하며 수정 제안

---

### 파일 트리 (File Trees)

- **설명**: 폴더와 파일의 계층 구조를 나무 모양으로 보여주는 목록이다.
- **use-when**:
  - 코드 에디터, 문서 관리 시스템에서 파일 계층 구조를 탐색할 때
  - 관리자 페이지에서 폴더 기반 콘텐츠를 구조적으로 보여줄 때
  - 사용자가 복잡한 계층형 데이터를 직관적으로 탐색해야 할 때
- **avoid-when**:
  - 계층 구조가 2단계 이하로 단순한 경우 (일반 목록이나 Accordion으로 충분)
  - 모바일에서 계층이 깊어 탐색이 불편한 경우
- **pairs-with**: Sidebars, Accordions, Menus, Icons
- **shadcn-mapping**: Collapsible (중첩 구조로 구현) 또는 커스텀 구현
- **variants**:
  - 기본형: 들여쓰기 + 폴더/파일 아이콘
  - 체크박스형: 파일 선택이 가능한 체크박스 포함 트리
  - 드래그 앤 드롭형: 파일 이동이 가능한 인터랙티브 트리

---

### 파일 업로드 (File Uploads)

- **설명**: 사용자가 자신의 컴퓨터에 있는 파일을 서버로 올릴 수 있게 하는 영역이다.
- **use-when**:
  - 프로필 사진, 서류, 첨부 파일 등을 사용자가 직접 올려야 할 때
  - 드래그 앤 드롭으로 직관적인 파일 업로드 경험을 제공할 때
  - 업로드 진행률을 시각적으로 보여주어야 할 때
- **avoid-when**:
  - 단순 파일 선택만 필요할 때 과도한 드래그존 UI를 구성하는 경우
  - 허용 파일 형식 안내 없이 업로드 후 오류를 보여주는 경우
- **pairs-with**: Forms, Buttons, Progress, Alerts
- **shadcn-mapping**: 커스텀 구현 (input[type=file] + 드래그존 + Progress 조합)
- **variants**:
  - 드래그존형: 점선 테두리 영역에 파일을 드래그하는 방식
  - 버튼형: "파일 선택" 버튼 클릭으로 파일 탐색기 열기
  - 미리보기형: 이미지 업로드 시 썸네일 미리보기 제공

---

### 폼 (Forms)

- **설명**: 이름, 이메일, 내용 등 여러 정보를 한꺼번에 입력받는 양식들의 집합이다.
- **use-when**:
  - 회원가입, 문의하기, 주문서 등 다수의 필드를 한 번에 처리할 때
  - 유효성 검사와 에러 메시지가 필요한 구조화된 입력이 필요할 때
  - 서버로 데이터를 제출하는 모든 사용자 입력 흐름
- **avoid-when**:
  - 단 하나의 입력만 필요한 경우 (단독 Input + Button으로 충분)
  - 가입 없이 체험 가능한 경우 불필요하게 폼을 먼저 요구하는 경우
- **pairs-with**: Inputs, Buttons, Alerts, Checkboxes, Selects
- **shadcn-mapping**: Form (react-hook-form + zod 검증 통합)
- **variants**:
  - 단일 컬럼: 모바일 친화적인 위에서 아래로 나열
  - 다중 컬럼: 데스크탑에서 2열 배치로 공간 효율화
  - 스텝형: 여러 단계로 나뉜 멀티스텝 폼 (마법사 UI)

---

### 아이콘 (Icons)

- **설명**: 텍스트 대신 의미를 전달하는 작은 시각적 기호이다.
- **use-when**:
  - 메뉴 항목, 버튼 레이블의 의미를 시각적으로 보강할 때
  - 국제화 서비스에서 언어에 의존하지 않고 의미를 전달할 때
  - Features 섹션에서 각 항목의 특징을 상징하는 아이콘으로 구분할 때
- **avoid-when**:
  - 아이콘만 사용하여 의미가 불명확한 경우 (Tooltip으로 반드시 보완)
  - 일관성 없이 여러 스타일의 아이콘 라이브러리를 혼용하는 경우
- **pairs-with**: Buttons, Badges, Tooltips, Navigation Menus
- **shadcn-mapping**: 커스텀 구현 (lucide-react 라이브러리 권장)
- **variants**:
  - 아웃라인형: 선으로만 그린 아이콘 (가벼운 느낌)
  - 솔리드형: 채워진 아이콘 (강조, 활성 상태)
  - 컬러형: 브랜드 컬러가 적용된 강조 아이콘

---

### 입력창 (Inputs)

- **설명**: 사용자가 텍스트를 직접 타이핑하여 입력할 수 있는 빈 칸이다.
- **use-when**:
  - 로그인 아이디/비밀번호, 이름, 이메일 등 단일 텍스트 입력
  - 검색창처럼 즉각적인 입력 반응이 필요한 곳
  - 폼의 가장 기본 구성 요소로 모든 텍스트 입력 필드
- **avoid-when**:
  - 긴 문장이나 여러 줄의 입력이 필요한 경우 (Text Area 사용)
  - 정해진 옵션 중 선택하면 되는 경우 (Select 또는 Radio Group 사용)
- **pairs-with**: Forms, Buttons, Labels, Dropdowns
- **shadcn-mapping**: Input
- **variants**:
  - 기본형: 단순 텍스트 입력
  - 아이콘형: 좌측 또는 우측에 아이콘 포함 (검색, 비밀번호 토글)
  - OTP형: 인증 코드 입력을 위한 분리된 단일 문자 칸

---

### 링크 (Links)

- **설명**: 클릭하면 다른 페이지나 외부 사이트로 연결되는 텍스트 요소이다.
- **use-when**:
  - 본문 내 참조 문서나 외부 리소스로 연결할 때
  - "비밀번호를 잊으셨나요?", "약관 보기" 등 부가 액션 제공
  - 푸터나 네비게이션에서 페이지 간 이동 경로를 텍스트로 제공할 때
- **avoid-when**:
  - 주요 행동 유도에 링크를 사용하는 경우 (Button 사용)
  - 링크인지 일반 텍스트인지 구분이 안 되는 스타일링
- **pairs-with**: Texts, Footers, Navigation Menus, Buttons
- **shadcn-mapping**: Button (variant: link)
- **variants**:
  - 인라인형: 본문 텍스트 색상으로 밑줄 표시
  - 외부 링크형: 새 탭 열기 아이콘 포함
  - 네비게이션형: 언더라인 없이 hover 시 색상 변경

---

### 메뉴 (Menus)

- **설명**: 이동할 수 있는 경로를 모아놓은 목록이다.
- **use-when**:
  - 우클릭 컨텍스트 메뉴로 상황별 옵션을 제공할 때
  - 사이드바나 헤더에서 계층적 네비게이션을 구성할 때
  - 복잡한 앱에서 메뉴바(파일/편집/보기)를 구성할 때
- **avoid-when**:
  - 선택지가 2~3개로 적은 경우 (Button 그룹이나 Toggle Group 사용)
  - 메뉴 항목 수가 너무 많아 스크롤이 필요한 경우 (검색 기능 추가 필요)
- **pairs-with**: Navigation Menus, Sidebars, Dropdowns, Icons
- **shadcn-mapping**: Menubar, ContextMenu, DropdownMenu
- **variants**:
  - 컨텍스트 메뉴형: 마우스 우클릭 시 위치에 나타나는 메뉴
  - 메뉴바형: 애플리케이션 상단의 파일/편집/보기 형태
  - 사이드 메뉴형: 사이드바 내 중첩 가능한 계층 메뉴

---

### 알림 (Notifications)

- **설명**: 실시간으로 발생하는 소식을 사용자에게 알리는 시스템 메시지이다.
- **use-when**:
  - 새 메시지 도착, 좋아요, 댓글 등 실시간 이벤트를 사용자에게 전달할 때
  - 업데이트 완료, 처리 결과 등 백그라운드 작업의 완료를 알릴 때
  - 읽지 않은 알림을 목록으로 모아 제공할 때
- **avoid-when**:
  - 사용자가 즉각 반응해야 하는 중요 알림을 자동 소멸 방식으로 처리하는 경우
  - 너무 빈번한 알림으로 사용자가 피로감을 느끼는 경우
- **pairs-with**: Badges, Toasts, Avatars, Menus
- **shadcn-mapping**: Sonner (toast) + 커스텀 알림 패널 (Sheet 또는 Popover)
- **variants**:
  - 벨 아이콘 드롭다운: 헤더 벨 아이콘 클릭 시 알림 목록 표시
  - 실시간 토스트: 이벤트 발생 시 화면에 팝업되는 즉시 알림
  - 알림 센터: 전체 알림 히스토리를 보여주는 전용 패널

---

### 숫자 입력 (Numbers)

- **설명**: 오직 숫자만 입력받거나, +/- 버튼으로 수량을 조절하는 컴포넌트이다.
- **use-when**:
  - 쇼핑몰 장바구니에서 상품 수량을 조절할 때
  - 나이, 인원수 등 정수 범위 내에서만 유효한 값을 입력할 때
  - 대시보드에서 목표 수치나 한도를 직접 설정할 때
- **avoid-when**:
  - 연속적인 범위 값 조절이 필요한 경우 (Slider 사용)
  - 소수점이 포함된 정밀 수치 입력 시 스텝 버튼이 불편한 경우
- **pairs-with**: Forms, Sliders, Buttons, Inputs
- **shadcn-mapping**: Input (type=number) + 커스텀 스텝 버튼 래퍼
- **variants**:
  - 스텝 버튼형: +/- 버튼으로 1씩 증가/감소
  - 인라인 편집형: 숫자 클릭 시 직접 타이핑 가능
  - 범위 제한형: min/max 값 도달 시 버튼 비활성화

---

### 페이지네이션 (Paginations)

- **설명**: 많은 양의 데이터를 여러 페이지로 나누어 보여주는 번호 매기기이다.
- **use-when**:
  - 게시판, 검색 결과, 주문 내역처럼 대량 데이터를 페이지 단위로 탐색할 때
  - 무한 스크롤보다 명확한 페이지 위치 파악이 중요한 경우
  - 특정 페이지로 바로 이동해야 하는 경우
- **avoid-when**:
  - 콘텐츠 피드처럼 자연스러운 스크롤이 더 적합한 경우 (무한 스크롤 사용)
  - 데이터가 20개 미만으로 페이지 분할이 불필요한 경우
- **pairs-with**: Tables, Cards, Selects (페이지당 항목 수)
- **shadcn-mapping**: Pagination
- **variants**:
  - 번호형: [이전] [1] [2] [3] ... [다음] 형태
  - 간소형: [이전] / [다음] 버튼만 있는 최소형
  - 점프형: 페이지 번호를 직접 입력할 수 있는 Input 포함

---

### 팝오버 (Popovers)

- **설명**: 특정 요소를 클릭하거나 마우스를 올릴 때 그 옆에 작은 설명 창이 뜨는 방식이다.
- **use-when**:
  - 아이디나 이름 위에 마우스를 올렸을 때 간이 프로필을 보여줄 때
  - 복잡한 필터나 날짜 선택기를 트리거 버튼 옆에 펼쳐 보여줄 때
  - Tooltip보다 더 많은 정보(버튼, 링크 등)를 포함한 미니 패널이 필요할 때
- **avoid-when**:
  - 단순한 텍스트 설명만 필요한 경우 (Tooltip으로 충분)
  - 모바일 터치 환경에서 hover 트리거 팝오버 사용 (클릭 트리거로 변경)
- **pairs-with**: Avatars, Date Pickers, Buttons, Tooltips
- **shadcn-mapping**: Popover
- **variants**:
  - 정보형: 읽기 전용 사용자 정보나 미리보기 카드
  - 액션형: 버튼이나 링크가 포함된 미니 액션 패널
  - 필터형: 드롭다운 필터나 날짜 선택기를 팝오버 안에 포함

---

### 라디오 그룹 (Radio Groups)

- **설명**: 여러 선택지 중 단 하나만 선택할 수 있는 동그란 버튼 모음이다.
- **use-when**:
  - 결제 수단, 배송 방법 등 반드시 하나만 선택해야 하는 경우
  - 선택지가 2~5개로 적어 모두 한 번에 보여줄 수 있는 경우
  - 설정 페이지에서 뮤추얼 익스클루시브한 옵션을 선택할 때
- **avoid-when**:
  - 선택지가 6개 이상으로 많은 경우 (Select로 대체)
  - 여러 개를 동시에 선택할 수 있어야 하는 경우 (Checkbox 사용)
- **pairs-with**: Forms, Checkboxes, Selects, Buttons
- **shadcn-mapping**: RadioGroup
- **variants**:
  - 수직 나열형: 위에서 아래로 옵션 나열
  - 수평 나열형: 좌우로 옵션 나열 (선택지가 짧은 경우)
  - 카드 선택형: 라디오 버튼 대신 전체 카드를 선택 영역으로 사용

---

### 선택 박스 (Selects)

- **설명**: 드롭다운과 유사하며, 미리 정의된 옵션 리스트에서 값을 선택하는 창이다.
- **use-when**:
  - 국가, 시/도, 언어 등 선택지가 많은 경우 드롭다운으로 제공
  - 폼 내에서 미리 정의된 옵션 중 하나를 선택해야 하는 필드
  - 정렬 기준, 필터 옵션 등 설정값을 선택할 때
- **avoid-when**:
  - 선택지가 3개 이하로 적은 경우 (Radio Group이 더 직관적)
  - 검색 기능이 필요한 대량의 옵션 (Command/Combobox 사용)
- **pairs-with**: Forms, Radio Groups, Tables, Paginations
- **shadcn-mapping**: Select
- **variants**:
  - 기본형: 단일 선택 드롭다운
  - 다중 선택형: 여러 값을 선택할 수 있는 멀티셀렉트
  - 검색형: 옵션을 타이핑으로 검색하는 Combobox

---

### 사이드바 (Sidebars)

- **설명**: 화면의 왼쪽이나 오른쪽에 고정된 세로형 메뉴 영역이다.
- **use-when**:
  - 대시보드나 관리자 시스템에서 주 네비게이션 메뉴로 사용
  - 복잡한 설정 페이지에서 카테고리별 섹션 이동에 활용
  - 이메일, 슬랙 등 좌측 메뉴가 항상 보여야 하는 앱 레이아웃
- **avoid-when**:
  - 모바일에서 항상 표시하여 메인 콘텐츠 영역을 좁히는 경우 (접을 수 있도록 구현)
  - 페이지가 2~3개뿐인 단순한 사이트 (Navigation Menu로 충분)
- **pairs-with**: Navigation Menus, Menus, Accordions, Badges
- **shadcn-mapping**: Sidebar
- **variants**:
  - 고정형: 항상 펼쳐진 상태로 고정
  - 접힘형: 아이콘만 보이는 축소 상태와 완전히 펼친 상태 토글
  - 오버레이형: 모바일에서 햄버거 버튼 클릭 시 오버레이로 등장

---

### 로그인/회원가입 (Sign Ins / Sign Ups)

- **설명**: 사용자 인증을 위한 전용 폼 페이지나 섹션이다.
- **use-when**:
  - 서비스 접근 제어가 필요한 모든 유료 또는 개인화 서비스
  - 소셜 로그인(Google, GitHub 등)을 함께 제공하여 진입 장벽을 낮출 때
  - 가입 완료 후 자연스러운 온보딩 플로우로 연결할 때
- **avoid-when**:
  - 탐색 단계의 사용자에게 가입을 강제하여 이탈을 유발하는 경우
  - 로그인 후 이전 페이지로 리다이렉트 처리를 생략하는 경우
- **pairs-with**: Forms, Inputs, Buttons, Alerts
- **shadcn-mapping**: Form + Input + Button + Separator (소셜 구분선) 조합
- **variants**:
  - 단순형: 이메일 + 비밀번호 + 로그인 버튼
  - 소셜 로그인형: Google/GitHub 버튼 + 이메일 폼 혼합
  - 분할 레이아웃형: 좌측 브랜드 비주얼 + 우측 로그인 폼

---

### 슬라이더 (Sliders)

- **설명**: 조절 바를 좌우로 움직여서 수치를 변경하는 컴포넌트이다.
- **use-when**:
  - 가격 범위, 음량, 밝기 등 연속적인 수치를 직관적으로 조절할 때
  - 사용자가 정확한 숫자보다 범위나 상대적 크기가 중요한 설정을 조절할 때
  - 필터에서 최솟값과 최댓값을 양 끝에서 조절하는 범위 슬라이더가 필요할 때
- **avoid-when**:
  - 정확한 숫자 입력이 중요한 경우 (Numbers Input과 병행 사용)
  - 이산적인 선택지가 몇 개 없는 경우 (Radio Group 사용)
- **pairs-with**: Forms, Numbers, Inputs, Cards
- **shadcn-mapping**: Slider
- **variants**:
  - 단일 핸들: 단일 값을 조절하는 기본 슬라이더
  - 범위형: 최솟값과 최댓값을 동시에 조절하는 듀얼 핸들
  - 단계형: 특정 값에만 정지하는 스텝 슬라이더

---

### 로딩 스피너 (Spinner Loaders)

- **설명**: 데이터가 처리 중임을 알려주는 회전하는 애니메이션 아이콘이다.
- **use-when**:
  - 데이터 로딩, 파일 업로드, API 응답 대기 중 처리 상태를 표시할 때
  - 버튼 클릭 후 처리 중 상태를 버튼 내부 스피너로 표시할 때
  - 전체 페이지 초기 로딩 시 빈 화면을 대체할 때
- **avoid-when**:
  - 로딩 시간이 길 때 스피너만 보여주는 경우 (Skeleton으로 레이아웃 유지)
  - 이미 완료된 상태에서 스피너가 계속 표시되는 경우
- **pairs-with**: Buttons, Skeleton, Alerts, Tables
- **shadcn-mapping**: Skeleton (레이아웃 로딩), 또는 커스텀 Spinner SVG 애니메이션
- **variants**:
  - 인라인형: 버튼 또는 아이콘 크기의 작은 스피너
  - 전체화면형: 페이지 로딩 시 중앙에 표시되는 큰 스피너
  - 오버레이형: 콘텐츠 위에 반투명 레이어와 함께 표시

---

### 표 (Tables)

- **설명**: 격자 형태로 데이터를 나열하여 비교하기 쉽게 만든 컴포넌트이다.
- **use-when**:
  - 거래 내역, 사용자 목록, 주문 관리 등 정형 데이터를 행열로 표시할 때
  - 정렬, 필터, 페이지네이션이 필요한 대량 데이터 관리 화면
  - 여러 항목의 속성을 나란히 비교해야 하는 경우
- **avoid-when**:
  - 모바일에서 많은 열(column)이 있어 가로 스크롤이 과도하게 발생하는 경우
  - 3개 이하의 간단한 데이터를 Cards로도 충분히 표현 가능한 경우
- **pairs-with**: Paginations, Checkboxes, Badges, Dropdowns, Empty States
- **shadcn-mapping**: DataTable (Table + TanStack Table 통합)
- **variants**:
  - 기본형: 단순 데이터 출력 테이블
  - 인터랙티브형: 정렬, 필터, 행 선택, 인라인 편집 기능 포함
  - 고정 헤더형: 스크롤 시 헤더가 고정되는 긴 데이터 테이블

---

### 탭 (Tabs)

- **설명**: 같은 영역에서 서로 다른 카테고리의 내용을 교체하며 보여주는 메뉴이다.
- **use-when**:
  - 마이페이지처럼 주문 내역/리뷰/개인 정보 등 관련 카테고리를 전환할 때
  - 동일 컨텍스트 내에서 다른 데이터 뷰를 제공해야 할 때
  - 코드 예제에서 언어별 또는 프레임워크별 코드를 탭으로 구분할 때
- **avoid-when**:
  - 탭이 5개 이상으로 많아 한 줄에 들어가지 않는 경우 (Dropdown이나 Sidebar로 대체)
  - 탭 내용 간에 흐름이 있어 순서대로 봐야 하는 경우 (Steps/Wizard UI 사용)
- **pairs-with**: Tables, Cards, Forms, Badges
- **shadcn-mapping**: Tabs
- **variants**:
  - 언더라인형: 선택된 탭에 밑줄 강조 (기본)
  - 필 형: 선택된 탭의 배경을 채워 구분
  - 세그먼트 컨트롤형: 전체를 둘러싼 박스 안에서 탭 전환

---

### 태그 (Tags)

- **설명**: 키워드를 강조하거나 분류할 때 사용하는 작은 캡슐 형태의 요소이다.
- **use-when**:
  - 게시물이나 제품에 카테고리/해시태그를 붙여 필터링을 지원할 때
  - 사용자 관심사나 기술 스택을 시각적으로 나열할 때
  - 검색 필터에서 선택된 조건을 태그 형태로 표시하고 X 버튼으로 제거할 때
- **avoid-when**:
  - 태그가 너무 많아 줄 바꿈이 과도하게 발생하는 경우 (최대 표시 수 제한)
  - 상태나 숫자를 나타내야 하는 경우 (Badge 사용)
- **pairs-with**: Inputs, Badges, Cards, Tables
- **shadcn-mapping**: Badge (variant 조합으로 태그처럼 스타일링)
- **variants**:
  - 읽기 전용형: 클릭 불가능한 단순 라벨 태그
  - 삭제 가능형: X 버튼이 있어 제거 가능한 인터랙티브 태그
  - 클릭 필터형: 클릭 시 필터가 적용되는 토글 가능 태그

---

### 텍스트 영역 (Text Areas)

- **설명**: 여러 줄의 긴 글을 입력받을 수 있는 커다란 입력창이다.
- **use-when**:
  - 문의 내용, 댓글, 제품 리뷰 등 자유 형식의 긴 텍스트를 입력받을 때
  - 코드 스니펫이나 JSON 등 멀티라인 텍스트 입력이 필요할 때
  - 메모나 메시지 작성처럼 가변 길이 텍스트를 작성할 때
- **avoid-when**:
  - 단일 줄의 짧은 입력이면 충분한 경우 (Input 사용)
  - 리치 텍스트(Bold, Italic, 링크 등)가 필요한 경우 (에디터 컴포넌트 사용)
- **pairs-with**: Forms, Buttons, Inputs, Labels
- **shadcn-mapping**: Textarea
- **variants**:
  - 고정 높이형: 행 수를 고정한 기본 textarea
  - 자동 확장형: 내용에 따라 높이가 자동으로 늘어나는 auto-resize
  - 문자 수 제한형: 최대 입력 문자 수와 현재 카운트를 표시

---

### 토스트 (Toasts)

- **설명**: 화면 하단에서 슬쩍 올라왔다가 자동으로 사라지는 짧은 알림 메시지이다.
- **use-when**:
  - 링크 복사, 저장 완료, 전송 성공 등 즉각적이고 비방해적인 피드백
  - 백그라운드 작업(파일 업로드, 데이터 처리) 완료를 조용히 알릴 때
  - 실행 취소 가능한 작업 후 "취소" 옵션을 짧은 시간 동안 제공할 때
- **avoid-when**:
  - 사용자가 반드시 확인해야 하는 중요 알림 (Alert나 Dialog 사용)
  - 에러 메시지처럼 사용자가 읽고 대응해야 하는 경우 (자동 소멸 부적합)
- **pairs-with**: Buttons, Forms, Alerts, Notifications
- **shadcn-mapping**: Sonner
- **variants**:
  - 성공형: 초록 아이콘 + 완료 메시지
  - 에러형: 빨간 아이콘 + 간단한 오류 설명
  - 실행 취소형: 메시지 + "취소" 버튼이 함께 있는 액션 토스트

---

### 토글/스위치 (Toggles)

- **설명**: 켜기/끄기(On/Off) 상태를 전환하는 스위치 버튼이다.
- **use-when**:
  - 다크모드, 알림 수신, 공개/비공개 설정 등 즉각적인 On/Off 전환
  - 설정 페이지에서 기능의 활성화 여부를 조절할 때
  - 필터나 옵션의 활성 상태를 시각적으로 명확히 보여줄 때
- **avoid-when**:
  - On/Off 이외에 3개 이상의 상태가 있는 경우 (Radio Group 사용)
  - 즉각 반영이 아닌 저장 버튼을 눌러야 적용되는 설정 (Checkbox + 저장 버튼 사용)
- **pairs-with**: Forms, Cards, Settings 페이지
- **shadcn-mapping**: Switch
- **variants**:
  - 기본 스위치형: iOS 스타일의 토글 슬라이더
  - 아이콘 토글형: 특정 기능을 아이콘 버튼으로 활성/비활성 토글
  - 그룹 토글형: 여러 옵션 중 하나 또는 여러 개를 활성화하는 Toggle Group

---

### 툴팁 (Tooltips)

- **설명**: 마우스를 올렸을 때 해당 요소에 대한 짧은 설명을 보여주는 말풍선이다.
- **use-when**:
  - 아이콘만 있는 버튼에 의미를 설명하는 레이블이 필요할 때
  - 용어, 약어, 기술적 개념에 대한 간단한 정의를 제공할 때
  - 장애가 있는 요소(disabled)에 비활성화 이유를 설명할 때
- **avoid-when**:
  - 중요한 정보를 툴팁에만 넣어 모바일 사용자가 접근하지 못하는 경우
  - 설명이 길어 툴팁 안에서 읽기 불편한 경우 (Popover 사용)
- **pairs-with**: Icons, Buttons, Badges, Tables
- **shadcn-mapping**: Tooltip
- **variants**:
  - 기본형: 검정 배경 흰 텍스트의 작은 말풍선
  - 리치형: 아이콘과 링크가 포함된 약간 큰 툴팁
  - 방향형: 요소의 위/아래/좌/우에 위치하는 방향 지정 툴팁
